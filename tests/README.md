# 测试脚本目录结构

## 目录说明

### `/api-tests/` - API 测试
- 测试后端 API 接口功能
- 包含各种 API 调用和响应验证

### `/browser-tests/` - 浏览器测试  
- 使用 Playwright/Chrome 进行的浏览器自动化测试
- 包含页面交互和导航测试

### `/mcp-tests/` - MCP 相关测试
- Model Context Protocol 相关的测试脚本
- 包含 MCP 连接、参数格式、工具调用等测试

### `/config-tests/` - 配置测试
- 配置管理和同步相关的测试
- 包含前端配置、后端配置验证

### `/debug-scripts/` - 调试脚本
- 用于问题诊断和调试的工具脚本
- 包含日志分析、执行流程跟踪等

### `/verification-scripts/` - 验证脚本
- 用于验证修复和功能的脚本
- 包含各种验证和检查工具

### `/integration-tests/` - 集成测试
- 端到端的集成测试脚本
- 包含完整的用户流程测试

## 使用说明

每个目录下的脚本都有特定的用途，运行前请确保：
1. 服务器已启动 (localhost:3001)
2. 必要的依赖已安装
3. 环境配置正确
