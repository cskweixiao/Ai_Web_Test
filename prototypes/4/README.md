# 功能测试用例列表原型设计

## 📋 概述

本项目提供了4个不同版本的功能测试用例列表界面设计原型，每个版本都针对不同的使用场景和需求进行了优化。

## 🎨 原型版本

### 版本1：经典表格视图
**文件**: `version1-table.html`

**特点**:
- 📊 传统的表格展示方式
- ✅ 信息密度高，适合快速浏览和对比数据
- ✅ 支持多列筛选和排序
- ✅ 批量操作支持（复选框选择）
- ✅ 完整的分页功能

**适用场景**:
- 需要查看详细数据的场景
- 数据导出和批量操作
- 习惯传统表格界面的用户

---

### 版本2：树形层级视图 ⭐推荐
**文件**: `version2-tree.html`

**特点**:
- 🌳 三层结构：测试场景 → 测试点 → 测试用例
- ✅ 层次关系清晰，便于管理
- ✅ 可折叠展开的树形结构
- ✅ 支持批量操作
- ✅ 数据统计概览

**适用场景**:
- 需要按场景管理测试点和测试用例
- 层级关系复杂的项目
- 需要快速定位特定场景下的测试点和用例

---

### 版本3：卡片式视图
**文件**: `version3-card.html`

**特点**:
- 🎴 卡片网格布局
- ✅ 视觉层次丰富，美观度高
- ✅ 快速浏览和定位
- ✅ 数据统计仪表板
- ✅ 支持多种视图切换

**适用场景**:
- 注重视觉体验的场景
- 需要快速浏览测试用例概览
- 移动端和触屏设备

---

### 版本4：看板视图
**文件**: `version4-kanban.html`

**特点**:
- 📋 按状态分列展示（草稿、待审核、已发布、已归档）
- ✅ 可视化工作流
- ✅ 支持拖拽移动卡片
- ✅ 直观的状态管理
- ✅ 敏捷开发风格

**适用场景**:
- 敏捷开发团队
- 需要跟踪测试用例生命周期
- 需要可视化状态流转

## 🚀 如何使用

### 方式1：直接在浏览器中打开
1. 打开 `index.html` 查看原型导航页面
2. 点击任意版本卡片查看对应的原型

### 方式2：使用本地服务器（推荐）
```bash
# 在 prototypes 目录下运行以下命令之一：

# Python 3
python -m http.server 8000

# Python 2
python -m SimpleHTTPServer 8000

# Node.js (需要先安装 http-server)
npx http-server -p 8000

# PHP
php -S localhost:8000
```

然后在浏览器中访问：`http://localhost:8000`

## 💡 交互功能

### 🎯 核心操作（所有版本）

#### 1. 查看详情 👁️
- 点击查看按钮查看测试点/用例的完整信息
- 弹窗形式展示，包含所有关键字段
- 格式化显示，易于阅读

#### 2. 编辑功能 ✏️
- 完整的编辑表单，支持修改所有字段
  - 测试目的
  - 测试点名称
  - 测试步骤（支持多行）
  - 预期结果（支持多行）
  - 风险等级（高/中/低）
  - 优先级（高/中/低）
- 表单自动填充当前数据
- 实时保存并刷新列表
- 成功提示反馈

#### 3. 执行测试 ▶️
- 显示测试步骤和预期结果
- 记录执行结果：
  - ✅ 通过
  - ❌ 失败
  - 🚫 阻塞
  - ⏭️ 跳过
- 添加执行备注
- 上传截图附件
- 提交后即时反馈

#### 4. 删除功能 🗑️
- 二次确认对话框
- 显示要删除的项目名称
- 警告提示：⚠️ 此操作不可恢复
- 删除成功后有Toast通知
- 自动刷新列表

#### 5. 批量操作（版本1）
- 复选框多选
- 批量删除功能
- 实时显示选中数量
- 一键清空选择

### 🎨 视觉反馈

#### Toast 通知系统
- 自动显示在右上角
- 3秒后自动消失
- 颜色区分操作类型：
  - 🟢 成功：绿色
  - 🔴 错误：红色
  - 🟠 警告：橙色
- 平滑的滑入/滑出动画

#### 弹窗系统
- 半透明遮罩背景
- 从底部平滑滑入
- 点击外部关闭
- ESC键快速关闭
- 自适应内容高度
- 响应式设计

#### 按钮交互
- 悬停时颜色变化
- 悬停时轻微放大
- 点击时按下效果
- 不同操作用不同颜色：
  - 执行：绿色
  - 删除：红色
  - 常规：灰色

### ⌨️ 键盘快捷键
- `ESC`: 关闭所有打开的弹窗
- `Enter`: 在搜索框中提交搜索
- `双击卡片` (看板视图): 快速查看详情

## 🎯 推荐选择

根据您的需求选择合适的版本：

1. **如果需要处理大量数据和批量操作** → 选择版本1（表格视图）
2. **如果需要清晰的层级管理** → 选择版本2（树形视图）⭐推荐
3. **如果注重视觉体验和快速浏览** → 选择版本3（卡片视图）
4. **如果采用敏捷开发方式** → 选择版本4（看板视图）

## 📄 操作页面

### 创建测试用例 (`test-case-create.html`)
- 🎨 紫色渐变主题
- ✅ 单页面表单，一次性填写所有信息
- ✅ 支持选择已有场景/测试点或创建新的
- ✅ 动态添加/删除测试步骤
- ✅ 测试数据放在测试步骤上方

### 查看详情 (`test-case-detail.html`)
- 🎨 紫色渐变主题
- ✅ 完整展示测试用例所有信息
- ✅ 层级路径导航（场景→测试点→用例）
- ✅ 快速操作按钮（执行/编辑）

### 编辑测试用例 (`test-case-edit.html`)
- 🎨 橙色渐变主题
- ✅ 完整的表单编辑功能
- ✅ 动态添加/删除测试步骤
- ✅ 优先级和风险等级选择
- ✅ 测试数据放在测试步骤上方

### 执行测试 (`test-execute.html`)
- 🎨 绿色渐变主题
- ✅ **显示前置条件**
- ✅ **每个步骤可单独记录执行结果**（通过/失败）
- ✅ **实时进度跟踪**（X / 4 步骤）
- ✅ **最终结果汇总**（通过/失败/阻塞）
- ✅ 实时计时器
- ✅ 截图上传功能（支持多张）
- ✅ 自动定位到下一步

## 📝 注意事项

- 这些是静态HTML原型，数据为模拟数据
- 所有交互效果均为前端演示，不包含后端逻辑
- 可根据实际需求调整样式和功能
- 建议在Chrome、Firefox、Edge等现代浏览器中查看
- 所有操作页面的返回按钮统一放置在右上角
