# 功能测试用例列表原型设计

## 📋 概述

本项目提供了4个不同版本的功能测试用例列表界面设计原型，每个版本都针对不同的使用场景和需求进行了优化。

## 🎨 原型版本

### 版本1：经典表格视图
**文件**: `version1-table.html`

**特点**:
- 📊 传统的表格展示方式
- ✅ 信息密度高，适合快速浏览和对比数据
- ✅ 支持多列筛选和排序
- ✅ 批量操作支持（复选框选择）
- ✅ 完整的分页功能

**适用场景**:
- 需要查看详细数据的场景
- 数据导出和批量操作
- 习惯传统表格界面的用户

---

### 版本2：树形层级视图 ⭐推荐
**文件**: `version2-tree.html`

**特点**:
- 🌳 三层结构：测试场景 → 测试用例 → 测试点
- ✅ 层次关系清晰，便于管理
- ✅ 可折叠展开的树形结构
- ✅ 支持批量操作
- ✅ 数据统计概览

**适用场景**:
- 需要按场景管理测试用例
- 层级关系复杂的项目
- 需要快速定位特定场景的测试用例

---

### 版本3：卡片式视图
**文件**: `version3-card.html`

**特点**:
- 🎴 卡片网格布局
- ✅ 视觉层次丰富，美观度高
- ✅ 快速浏览和定位
- ✅ 数据统计仪表板
- ✅ 支持多种视图切换

**适用场景**:
- 注重视觉体验的场景
- 需要快速浏览测试用例概览
- 移动端和触屏设备

---

### 版本4：看板视图
**文件**: `version4-kanban.html`

**特点**:
- 📋 按状态分列展示（草稿、待审核、已发布、已归档）
- ✅ 可视化工作流
- ✅ 支持拖拽移动卡片
- ✅ 直观的状态管理
- ✅ 敏捷开发风格

**适用场景**:
- 敏捷开发团队
- 需要跟踪测试用例生命周期
- 需要可视化状态流转

## 🚀 如何使用

### 方式1：直接在浏览器中打开
1. 打开 `index.html` 查看原型导航页面
2. 点击任意版本卡片查看对应的原型

### 方式2：使用本地服务器（推荐）
```bash
# 在 prototypes 目录下运行以下命令之一：

# Python 3
python -m http.server 8000

# Python 2
python -m SimpleHTTPServer 8000

# Node.js (需要先安装 http-server)
npx http-server -p 8000

# PHP
php -S localhost:8000
```

然后在浏览器中访问：`http://localhost:8000`

## 💡 交互功能

所有原型都包含以下交互功能：

- ✅ 搜索和筛选
- ✅ 数据展示
- ✅ 操作按钮（查看、编辑、删除）
- ✅ 响应式布局
- ✅ 悬停效果和过渡动画

特殊交互：
- **版本2**: 支持树形节点的展开/折叠
- **版本3**: 卡片悬停放大效果
- **版本4**: 支持拖拽卡片到不同状态列

## 🎯 推荐选择

根据您的需求选择合适的版本：

1. **如果需要处理大量数据和批量操作** → 选择版本1（表格视图）
2. **如果需要清晰的层级管理** → 选择版本2（树形视图）⭐推荐
3. **如果注重视觉体验和快速浏览** → 选择版本3（卡片视图）
4. **如果采用敏捷开发方式** → 选择版本4（看板视图）

## 📝 注意事项

- 这些是静态HTML原型，数据为模拟数据
- 所有交互效果均为前端演示，不包含后端逻辑
- 可根据实际需求调整样式和功能
- 建议在Chrome、Firefox、Edge等现代浏览器中查看
